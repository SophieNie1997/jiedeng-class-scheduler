# 学生请假与待补课设计

## 背景

当前排课老师遇到学生生病、临时有事等请假场景时，只能删除对应课节。删除可以让课表变干净，但会丢失一个重要事实：这节课原本存在，只是没有实际发生，并且后续可能需要补课。

新设计把“学生请假”从“删除课程”里拆出来，保留原课节记录，同时让排课老师能看到原时间点发生了请假，并从一个集中入口跟进待补课。

## 目标

1. 课程详情里提供比删除更准确的“标记请假”操作。
2. “所有老师总课表”的月视图和周视图都保留一个轻量请假标记，让原本的时间位置有迹可循。
3. 页面顶部提供“待补课”入口，集中查看所有请假后尚未补课的课节。
4. 请假课不计入老师课时统计，也不显示成正常课程色块。
5. 请假标记可以撤回，误操作后能恢复为正常课。

## 非目标

第一版不自动寻找补课时间，不做家长通知，不做审批流，不做课时包扣费规则。补课安排先沿用现有“新增课程”能力，后续可以再扩展成从待补课列表一键创建补课。

## 数据模型

现有课程编辑集中保存在 `lessonEdits.updates`，删除保存在 `lessonEdits.deletedIds`。请假应使用更新记录保存，不进入删除列表。

对请假课程保存以下字段：

- `status`: `请假`
- `absenceStatus`: `待补课` 或 `已补课`
- `absenceReason`: 请假原因，例如 `生病`、`临时有事`、`其他`
- `absenceNote`: 可选备注
- `absenceMarkedAt`: 标记时间，便于后续追踪

撤回请假时，保留课程原有日期、老师、学生、课程、时间等信息，将状态恢复为普通编辑状态，并清除请假字段。

## 课程详情

课程详情保留现有保存和删除能力，同时增加一个次级操作 `标记请假`。

点击 `标记请假` 后弹出确认对话框，默认原因是 `生病`，也允许选择 `临时有事` 或填写备注。确认后课程状态变为 `请假`，并立即保存同步。

当课程已经是请假状态时，详情页显示当前原因和两个操作：

- `恢复为正常课程`
- `标记已补课`

`恢复为正常课程` 用于误操作撤回。`标记已补课` 用于把这条记录从待补课列表中移出，但仍保留历史请假状态。

## 日历显示

“所有老师总课表”的月视图和周视图都不把请假课渲染成正常课程卡片。

在原日期、原时段位置显示一个轻量请假标记，内容包含：

- `请假`
- 时间
- 学生和课程的短文本

视觉上使用淡灰或淡粉的小标记，与正常课程的老师/课程色块明显区分。请假标记可点击，点击后打开同一课程详情。

如果某个时段只有请假课，日期格仍能看出这里原本有安排，但不会把它误读为已上课。

## 待补课入口

在“所有老师总课表”顶部控制区增加 `待补课` 入口，靠近 `课时统计` 或作为同组按钮。

点击后在当前页面展开一个表格，列出所有 `status = 请假` 且 `absenceStatus = 待补课` 的课节。表格字段：

- 原日期
- 原时间
- 老师
- 学生
- 课程
- 请假原因/备注
- 操作

操作包含：

- `查看详情`
- `恢复正常`
- `标记已补课`

列表默认按原日期倒序或最近待处理优先排序；排课老师能快速看到还有哪些课需要安排补课。

## 统计规则

`课时统计` 不计算 `status = 请假` 的课节，因为这节课没有实际发生。

如果后续新增了一节补课，补课作为新的正常课程计入统计。原请假记录保持不计入统计，只用于追踪待补课。

日历上的每日课程数量也不把请假课计为正常课节数；如需显示，可单独显示 `1 个请假` 这类弱提示。

## 错误处理与同步

请假状态和普通课程编辑一样，先写入本地状态，再通过现有 Supabase 同步链路保存。只读模式下，`标记请假`、`恢复正常`、`标记已补课` 与其他写操作一样被拦截。

如果同步失败，本地仍保留修改，并显示现有的同步失败提示。

## 测试计划

新增测试覆盖以下行为：

1. `lessonEdits` 能保存请假字段，并且不会把请假课加入删除列表。
2. 日历汇总函数能识别请假课，并把它从正常课程统计中排除。
3. 月视图和周视图源码包含请假标记渲染入口。
4. `课时统计` 不计算请假课。
5. 待补课列表只显示 `请假 + 待补课` 的课程，不显示 `已补课` 或普通课程。
6. 只读模式下，请假相关写操作走现有写保护逻辑。

## 实施顺序

1. 增加请假状态的常量和判断函数。
2. 扩展课程详情操作区，加入请假确认和撤回/已补课操作。
3. 调整日历渲染，让请假课显示为轻量标记。
4. 新增待补课面板和列表操作。
5. 调整课时统计和日期课节计数，排除请假课。
6. 增加缓存版本号并完成本地与线上验证。
